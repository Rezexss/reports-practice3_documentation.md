# Название проекта: GameDistribution Platform (учебный проект, аналог Steam)

## Цели проекта
Разработка и документирование учебной модели клиент-серверной платформы для цифрового распространения игр. Основные цели:
1.  Изучение принципов построения систем аутентификации и управления пользователями.
2.  Освоение процессов резервного копирования и восстановления данных пользователей.
3.  Получение навыков написания технической и пользовательской документации на примере реально существующего сервиса.

## Описание практических работ

### Практическая работа №1: Настройка окружения и структуры проекта
*   **Цель:** Инициализация репозитория, создание структуры папок, проектирование схемы базы данных для хранения информации об играх и пользователях.
*   **Результат:** Создан локальный git-репозиторий, настроен удаленный репозиторий на GitHub, разработана структура таблиц `users`, `games`, `licenses`.

### Практическая работа №2: Разработка функций сопровождения
*   **Цель:** Реализация функций резервного копирования и восстановления данных, написание скриптов для автоматизации этих процессов.
*   **Результат:** Добавлены скрипты `backup.sh` и `restore.sh` для дампа базы данных, обновлена документация по их использованию.

### Практическая работа №3: Документирование и оформление итогового отчёта
*   **Цель:** Оформление технической документации и итогового отчета по сопровождению системы.
*   **Результат:** Создана инструкция пользователя (вход/выход/покупка игры), составлено техническое описание модулей и структуры системы (по аналогии со Steam), актуализирован данный файл `README.md`.

## Установка и запуск (учебного прототипа)
*Данный раздел описывает запуск учебного API, имитирующего функционал платформы.*

1.  Клонируйте репозиторий: `git clone <URL>`
2.  Установите зависимости: `pip install -r requirements.txt` (для Python) или `npm install` (для Node.js)
3.  Настройте подключение к базе данных в файле `.env`.
4.  Запустите миграции для создания таблиц: `python migrate.py`
5.  Запустите сервер: `python app.py`
6.  API будет доступно по адресу: `http://localhost:5000`
